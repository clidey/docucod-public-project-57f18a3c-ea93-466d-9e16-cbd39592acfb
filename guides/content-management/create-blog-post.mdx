# Create and Publish a Blog Post

Empower your personal website by adding fresh, insightful blog content. This guide walks you through the entire process of creating and publishing blog posts using Markdown, helping you manage content effectively and showcase your expertise with ease.

---

## Workflow Overview

### Task Description
Learn how to draft, format, and publish blog articles in your Astro-powered portfolio site, using Markdown files organized in your project structure.

### Prerequisites
- Your portfolio site must be set up and running locally.
- Basic understanding of Markdown syntax.
- Access to the project folder where the blog content resides (`src/content/blog`).

### Expected Outcome
By following this guide, you will:
- Create a new blog post file with proper metadata.
- Write and format your blog content using Markdown/MDX.
- Preview the post locally with consistent site styling.
- Have the new post automatically included in your site's blog index, sitemap, and RSS feed.

### Time Estimate
Allow approximately 15–30 minutes for your first post creation and preview.

### Difficulty Level
Beginner to Intermediate

---

## Step-by-Step Instructions

### Step 1: Create Your Blog Post File
1. Navigate to the `src/content/blog` directory in your project.
2. Create a new Markdown (`.md`) or MDX (`.mdx`) file with a clear, URL-friendly name (e.g., `my-first-post.md`).
   - **Best Practice**: Use lowercase letters, hyphens instead of spaces, and avoid special characters.

### Step 2: Add Frontmatter Metadata
At the top of your Markdown file, include frontmatter tags enclosed by `---` delimiters. Example:

```markdown
---
title: "My First Blog Post"
description: "An introduction to writing blog posts with Astro."
pubDate: "2024-06-10"
tags: [astro, blogging]
---
```

- **title**: Your blog post's main title.
- **description**: Brief summary for SEO and previews.
- **pubDate**: Publication date (YYYY-MM-DD format recommended).
- **tags**: (Optional) Relevant keywords or categories.

### Step 3: Write and Format Your Content
Start writing your article using Markdown syntax:
- Use headings (`#`, `##`, etc.) to structure your post.
- Add images, links, or code blocks using standard Markdown.
- Optionally, embed interactive elements with MDX if your file extension is `.mdx`.

Example:

```markdown
## Introduction

Welcome to my blog where I share insights about Astro.

```javascript
console.log('Hello, Astro!');
```

![Sample Image](/assets/images/sample.png)
```

### Step 4: Save and Preview Locally
1. Start or restart your development server:
   ```bash
   npm run dev
   ```
2. Open your browser and navigate to `/blog`.
3. Find your new post listed by its publication date.
4. Click on the post title to preview it in your site’s layout, ensuring:
   - Metadata like the title and publication date appear correctly.
   - Formatting and embedded content render as expected.

### Step 5: Manage Publication and Site Integration
- Your post automatically appears on the blog index page `/blog/index.html` sorted by date.
- The sitemap and RSS feed update dynamically to include your new post’s URL.
- No additional configuration is needed unless you want to customize metadata or sitemap settings via `astro.config.mjs`.

### Step 6: Optional Enhancements
- Add or update the hero image by including `heroImage` in frontmatter:

```yaml
heroImage: "/assets/images/blog-hero.jpg"
```

- Tag your posts for better organization and potential filtering in the future.

## Practical Tips & Best Practices
- **Consistent Naming:** Keep file names and paths consistent to avoid broken links.
- **Complete Metadata:** Always fill `title`, `description`, and `pubDate` for SEO benefits.
- **Use MDX Wisely:** Leverage MDX for rich interactive content but keep basic posts simple for performance.
- **Preview Frequently:** Reload and test your post on multiple devices to confirm responsive design and layout integration.

---

## Troubleshooting

### Common Issues and Solutions

- **Post Does Not Appear in Blog List:**
  - Ensure the file is saved within `src/content/blog`.
  - Confirm frontmatter contains a valid `pubDate`.
  - Restart your development server to refresh.

- **Formatting Issues or Broken Layout:**
  - Validate your Markdown syntax.
  - Check for missing closing tags if using MDX.
  - Ensure images are correctly referenced with relative paths.

- **Metadata Not Showing Correctly:**
  - Verify frontmatter keys and formats match expected schema.
  - Check the date format — should be valid ISO string or `YYYY-MM-DD`.

<Tip>
If you encounter deployment issues with new posts not showing after build, verify that your `astro.config.mjs` includes the blog URLs correctly in the sitemap integration.
</Tip>
